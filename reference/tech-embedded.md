## 1. MCU 与 RTOS

### 1.1 中断
- **中断上下文与进程上下文区别** — 中断上下文不可睡眠（无进程描述符/task_struct）、不可调用可能阻塞的函数（mutex/semaphore/内存分配 GFP_KERNEL，必须 GFP_ATOMIC）、不可访问用户态内存、执行时间必须极短（快速处理+推后下半部 Tasklet/Workqueue 处理耗时工作）
- **中断优先级与嵌套** — 高优先级中断可抢占低优先级正在执行的中断。需注意临界区保护（中断嵌套可能导致共享数据被高优先级中断操作打断/不一致→在该中断级别及以上屏蔽同/高级中断）。现代 RTOS 支持可配置最大嵌套深度
- **中断延迟与抖动** — 中断延迟：从硬件信号发出到 ISR 第一条指令执行的时间。影响因素：最长关中断时间（临界段长度 + 同级中断屏蔽）、指令流水线刷新开销。实时系统核心指标，纳秒级

### 1.2 RTOS 任务调度
- **FreeRTOS 调度策略** — 优先级的抢占式调度（高优任务就绪立刻抢占低优）、同优先级时间片轮转（configUSE_TIME_SLICING=1 时，同优先级任务按 tick 轮转）。Idle Task 释放删除任务的内存（vTaskDelete 使 TCB+stack 由 idle task 清理）
- **优先级反转与解决** — 低优先级任务持有资源（如 mutex）→ 高优先级任务阻塞等待该资源 → 中等优先级任务抢占 CPU 使低优先级任务无法释放资源 → 高优先级实际被中优先级阻塞。解决：优先级继承（持锁期间临时提升低优先级任务到等待该锁的最高优先级）→释放锁恢复原优先级
- **临界区与互斥** — 关中断（最简单/临界段极短/禁调度/高优中断响应受影响）；互斥信号量（mutex 带优先级继承/允许阻塞等待/适合临界段较长或可能阻塞场景）；自旋锁（SMP 多核/极短时间/忙等不切换上下文）。中断中只能用关中断/自旋锁，不能用 mutex

## 2. 嵌入式 Linux

### 2.1 内核与驱动
- **字符设备驱动框架** — struct file_operations（open/read/write/ioctl/release 等操作函数指针）→ 实现这些操作 → register_chrdev_region 或 alloc_chrdev_region 注册设备号 → cdev_init/cdev_add 注册字符设备到内核。用户空间通过 `/dev/xxx` open/read/write 调用驱动
- **设备树（DTS）** — 描述硬件拓扑与配置（而非硬编码在 C 源码中）。节点 = 设备 + compatible 字符串匹配到对应驱动。内核启动时解析 DTB（DTS 编译出的二进制 → .dtb），加载匹配驱动的 probe()。作用：同一内核镜像可适配不同硬件板（修改 DTS 不修改内核源码）
- **内核态与用户态数据交互** — copy_from_user/copy_to_user（必要时复制数据，保护内核免受用户态非法指针崩溃内核），get_user/put_user（简单类型值交换）。mmap 实现零拷贝（映射内核缓冲区到用户态地址空间，适合大量数据交互/驱动/视频采集）

### 2.2 系统移植
- **U-Boot 启动流程** — ROM Code → SPL（Secondary Program Loader，小体积加载到 SRAM）→ U-Boot proper（加载到 DRAM，初始化 DDR/Flash/网口等硬件 + 加载内核+DTB+根文件系统到内存 → bootm 跳转到内核入口）
- **文件系统选择** — ext4（通用/日志/可靠，适合 SD/eMMC 但写次数有限）；SquashFS（只读压缩/极适合只读根文件系统固件，压缩比高不可写）；UBIFS（为原始 NAND Flash 设计，均衡写/坏块管理/断电保护）；initramfs（内存中最小根文件系统，内核自带/恢复/维护模式用）

## 3. 通信协议

### 3.1 片上总线
- **I2C vs SPI vs UART 比较** — I2C 两线（SCL/SDA 带地址寻址+ACK 确认）、多主多从/速率较低（标准 100k/快速 400k/高速 3.4Mbps）。SPI 四线（MOSI/MISO/SCLK/SS 片选）、全双工/速率更高可达几十 Mbps/无标准协议/主机控制。UART 全双工/异步/需事先约定波特率数据格式/三线（TX/RX/GND）/点对点
- **CAN 总线** — 多主（无主机/从机概念，节点都可主动发送）、优先级仲裁（ID 越小优先级越高，碰撞时低优先级退避无损）、错误处理（CRC 校验+ACK 确认+错误计数跟踪故障节点）。汽车/工业领域主流

### 3.2 无线协议
- **BLE vs WiFi vs LoRa vs NB-IoT** — IoT 选型对比：BLE 极低功耗/短距 10-100m/1Mbps/手机直连，适合可穿戴+传感器小数据。WiFi 中距 100m/高吞吐几十 Mbps/高功耗，适合音视频/固件 OTA。LoRa 超远距km级/极低功耗/极低带宽<50kbps/星型网络，适合农村/郊区传感器。NB-IoT 蜂窝网络授权频谱/覆盖强/深度穿透/低功耗，适合广覆盖固定设备
- **OTA 升级方案** — 双分区（A/B 分区，下载新固件到非活动分区→校验→切换引导分区→重启。失败可回滚到旧分区，代价双倍存储）。单分区+恢复分区（小恢复分区引导+网络重拉完整固件自动修复，适合存储紧场景）。升级安全：固件签名+校验后再切换、HTTPS 下载+Hash 校验、防版本回退

## 4. 低功耗设计

- **MCU 功耗模式层次** — 运行（全速）→ Sleep（CPU 停/外设运行/中断唤醒）→ Deep Sleep（高频时钟停/低频振荡器跑/少数外设+唤醒源/内核状态丢失但 SRAM 保留）→ Shutdown（全部断电，仅有复位/唤醒引脚的极小功耗电路保持工作/SRAM 丢失。唤醒从复位向量开始运行）
- **功耗估算与优化** — 动态功耗 Pdyn = CV²f（C 负载电容 V 电压² f 频率→电压/频率大幅影响功耗）；静态功耗 Pstatic = V × Ileak（漏电流，随温度指数上升）。优化：降频/降核心电压、关不用的外设模块时钟（Clock Gating）、关不用的 IP 块电源（Power Gating 断电消漏电流）。中断驱动与事件唤醒将大部分时间休眠（占空比 < 1%）

## 5. 调试与测试工具

- **调试工具链** — J-Link/ST-Link（硬件调试器，支持断点/单步/寄存器查看/内存监控）；printf 调试（UART/RTT 串口，简单但影响实时性）；逻辑分析仪（解码 SPI/I2C/UART 时序，排查硬件通信故障）；示波器（电源纹波/信号完整性）
- **内存泄漏与栈溢出排查** — 栈溢出：栈 canary 检测（编译时插入哨兵值，破坏时 trap）；静态栈分析（编译期计算最大栈深度）；RTOS 栈水位标记（填充 0xAA 后检测使用量）。内存泄漏：heap 使用统计（malloc/free 计数配对）；Valgrind（Linux 下）；嵌入式场景用自定义 malloc 包装器追踪分配/释放
- **固件测试策略** — 单元测试（Unity/CMock/CppUTest 框架，在主机上运行，脱离硬件）；硬件在环测试 HIL（真实硬件运行测试固件，自动测试脚本验证功能）；Mock 硬件层（将硬件寄存器抽象为接口，主机测试时替换为 Mock 实现）

## 6. 安全与固件保护

- **固件安全启动** — BootROM 内置根证书 → 验证 Bootloader 签名 → Bootloader 验证 App 固件签名。信任链：任何一层被篡改都会启动失败。硬件支持：Secure Element（安全芯片存储密钥）/TrustZone（ARM 安全世界隔离）
- **固件防逆向** — 代码加密（Flash 加密，密钥在芯片内不可导出）；调试接口熔断（生产版本关闭 JTAG/SWD 调试口）；代码混淆（控制流平坦化/虚假分支）
